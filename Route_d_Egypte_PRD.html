# Product Requirements Document (PRD): Route d'Égypte Homepage Clone ## 1. Project Overview The goal is to recreate the high-end, elegant homepage of [Route d'Égypte](https://routedeegypte.com/) as a modern React application. The UI should convey luxury, trust, and the adventurous spirit of Egyptian travel. ## 2. Technical Stack - **Frontend:** React.js - **Routing:** React Router - **Styling:** CSS-in-JS or Tailwind (to be defined by Antigravity based on the prompt) - **Structure:** Component-based architecture for modularity and reuse. ## 3. Visual Identity (Based on Design System) - **Primary Palette:** Gold/Sand (#C79A3C), Deep Charcoal/Blue for text, and ample white space. - **Typography:** Sophisticated sans-serif (Plus Jakarta Sans) for modern legibility. - **Imagery:** High-resolution, warm-toned photography of landscapes and historical sites. - **Components:** Rounded corners (4px-8px), subtle drop shadows on cards, and elegant icon sets. ## 4. Page Structure & Sections ### 4.1. Header & Navigation - Transparent-to-solid sticky header. - Logo on the left, multi-level navigation in the center, and a clear CTA (Contact) on the right. - Language switcher. ### 4.2. Hero Section - Full-width background image with a subtle dark overlay. - Large, centered H1 heading: "Circuits et Excursions Privées en Égypte". - Subtitle and a primary "Call to Action" button. ### 4.3. Values / "Pourquoi Réserver Avec Nous" - Horizontal grid of icons (Specialization, Flexibility, Private Services, Ease, Reasonable Prices, Secure Payment). - Minimalist style with subtle hover effects. ### 4.4. Tour Categories (Croisières, Safari, Circuits, Excursions) - Standardized Card Component: Image with ribbon/label, Title, Duration, Description snippet, Price, and "Voir les détails" button. - Section headers with accent lines and descriptive subtitles. - "Voir Plus" buttons at the end of each category grid. ### 4.5. Planning / CTA Section - "Prêt à planifier votre voyage de rêve ?" banner. - Checklist of benefits. - Form or primary contact button. ### 4.6. Stats / Counter Section - Numerical stats (Clients, Tours, Reviews, etc.) with gold background. ### 4.7. Testimonials & Footer - Carousel or grid of user reviews with star ratings and source (TripAdvisor). - Footer: Multi-column layout with About Us, Quick Links, Destinations, and Social Media integration. --- # Antigravity Design Prompt **Task:** Build a high-fidelity React frontend clone of the Route d'Égypte homepage based on the following detailed specifications. **Architecture:** - Create a `src/components` folder for shared UI: `Header`, `Footer`, `TourCard`, `SectionHeader`, `Button`, `ReviewCard`. - Use `react-router-dom` for navigation (setup routes even if they point to placeholders). - Ensure the layout is fully responsive (Desktop focus for now, but mobile-ready grid). **Visual Style (Design System {{DATA:DESIGN_SYSTEM:DESIGN_SYSTEM_2}}):** - **Colors:** Use `#C79A3C` as the primary gold accent for buttons, icons, and highlights. Use a very light gray background for alternating sections. - **Typography:** Apply 'Plus Jakarta Sans' globally. Font weights should vary from 400 (body) to 700 (headings). - **Cards:** White background, 4px border-radius, `box-shadow: 0 4px 15px rgba(0,0,0,0.05)`. Image should have a slight zoom-on-hover effect. - **Buttons:** Solid gold background with white text for primary actions; outlined gold for secondary. **Section-by-Section Detail:** 1. **Header:** Height 80px. Logo should be a placeholder image with the text "Route d'Égypte". Links: Accueil, Croisières, Safari, Circuits, Excursions, Blog, Contact. 2. **Hero:** Full viewport width, 600px height. Use a placeholder URL for the desert landscape. Overlay text must be sharp and legible. 3. **The Card Grids:** Use a 3-column grid for desktop. Cards should include: - Top-right "Gold Ribbon" for category. - Title in bold 18px. - "Duration" icon (clock) and "Location" icon (map-pin). - Price in bold gold at the bottom right. 4. **Trust Section:** Use a 6-column grid for the "Why book with us" icons. 5. **Footer:** Dark background or heavy gold top-border. Four columns: Logo/Intro, Quick Links, Destinations, Contact Info. **Code Quality:** - Use functional components and hooks. - Use Semantic HTML (header, nav, main, section, footer). - Maintain clean, commented code for easy integration with MCP servers.
